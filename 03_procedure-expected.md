# 手順期待値出し

テスト設計の第3段階：パターン表の各行を、実際に実行可能なテストケースに落とし込む作業です。

## 目的

- パターン表の抽象的な記述を、具体的な操作手順に変換する
- 期待される結果を明確に記述する
- テスト実行者が迷わず実施できるレベルまで詳細化する

## テストケース（手順・期待値）の構成要素

### 事前条件

テストを実行する前に満たすべき状態。

例：

- ログイン済みであること
- BTCの残高が1.0以上あること

### テスト手順

ステップバイステップの操作内容。

例：

1. 出金画面を開く
2. 通貨で「BTC」を選択する
3. 出金先アドレスに「1A1zP1eP5QGefi2DMPTfTL5SLmv7DivfNa」を入力する
4. 出金額に「0.5」を入力する
5. 出金申請ボタンをクリックする

### 期待結果

各ステップ、または最終的に期待される画面表示やシステムの挙動。

例：

- 確認ダイアログが表示されること
- 「出金申請を受け付けました」というメッセージが表示されること
- 出金履歴一覧に新しいレコードが追加されること

## 作業の流れ

1. パターン表の各行を1つのテストケースとして展開
2. 事前条件を記述
3. 操作手順を番号付きリストで記述
4. 各ステップまたは最終結果として期待値を記述
5. テスト実行者がレビューし、曖昧な箇所を修正

# UIリグレッション確認チェックリスト

最終更新: 2026-03-08

このファイルは、画面クラッシュや導線不整合の再発を短時間で確認するための手動チェックリストです。

## 目的

- 注文直後に履歴画面へ遷移したときのクラッシュ再発を防ぐ
- APIレスポンスの欠損値があっても画面が落ちないことを確認する
- ログイン導線、通知表示、レイアウト崩れの回帰を確認する

## 事前準備

1. DBコンテナ起動を確認する

```powershell
docker compose ps
```

2. アプリを起動する

```powershell
.\start.bat
```

3. ブラウザで `http://localhost:5000/login` を開く

## スモークチェック（毎回実施）

1. ログイン画面で `開発者ログイン(User1)` が成功し、トップへ遷移する
2. ヘッダーにユーザー名と `ログアウト` が表示される
3. ログアウト後、`/login` に戻る
4. 画面表示中にコンソールで致命的エラーが出ない

## 回帰チェック項目

### 1. 注文直後に履歴へ遷移しても画面が消えない

1. 注文パネルで成行買い注文を実行する
2. 成功メッセージ表示中に `取引履歴` タブへ移動する
3. 注文履歴テーブルが表示され続けることを確認する
4. 画面全体が白画面にならないことを確認する

期待結果:
- クラッシュしない
- 注文履歴が表示される（新規注文が即時に出ない場合でも画面は維持される）

### 2. 欠損値がある注文データでも履歴描画が落ちない

1. `取引履歴` を開く
2. 以下の表示を確認する

期待結果:
- 日時が欠損時は `-` 表示
- `price`/`amount`/`total` 欠損時も数値フォーマットで例外にならない
- ステータス `open` と `pending` は `未約定` 表示

### 3. 注文成功メッセージの表示位置とアニメーション

1. 成行または指値注文を1件作成する
2. `BUY` ボタン直下に成功メッセージが出ることを確認する

期待結果:
- 表示時間は約5秒
- 最後の約0.5秒でフェードアウト

### 4. モード切替時のチャートX軸崩れ

1. チャート表示中に現物/レバレッジを切り替える
2. ウィンドウ幅を広い/狭いで切り替える

期待結果:
- X軸ラベルが見切れない
- チャートが横幅に追従して再描画される

### 5. 管理画面の401ハンドリング

1. 管理画面を開く
2. 無効トークン状態を作る（例: ログアウト後の再アクセス）

期待結果:
- `読み込み中...` 固定にならない
- ログイン画面に戻る

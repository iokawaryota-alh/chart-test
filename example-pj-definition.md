# プロジェクト定義書（仮）：暗号資産入出金機能

> **注意**: この定義書はトレーニング用に作成されたもので、意図的に不完全な箇所があります。
> 実務で使用する場合は、必ず不足要素を洗い出し、補完してください。

## 1. 概要

暗号資産取引プラットフォームにおいて、ユーザーが保有する暗号資産（BTC、ETH）の入金・出金機能を実装する。

## 2. 機能要件

### 2.1 入金機能

ユーザーが外部ウォレットから本プラットフォームのウォレットアドレスへ暗号資産を送金すると、入金処理を行う。

- 対応通貨：BTC、ETH
- 入金履歴を一覧で表示
- ステータス：処理中 / 完了

### 2.2 出金機能

ユーザーが指定した外部ウォレットアドレスへ暗号資産を送金する。

- 対応通貨：BTC、ETH
- 出金手数料
  - BTC：0.0005 BTC
  - ETH：0.005 ETH

## 3. 画面仕様

### 3.1 入金画面

表示項目：

- ユーザーの入金用ウォレットアドレス（コピーボタン付き）
- QRコード
- 入金履歴一覧
  - 日時
  - 通貨
  - 金額
  - ステータス（処理中 / 完了）

### 3.2 出金画面

入力フォーム：

- 通貨選択（ラジオボタン：BTC / ETH）
- 出金先アドレス（テキストボックス、必須）
- 出金額（テキストボックス、必須）

表示項目：

- 現在の残高
- 出金手数料
- 受取実額（出金額から手数料を差し引いた金額）

ボタン：

- 出金申請ボタン

画面動作：

- 出金額を入力すると、手数料と受取実額をリアルタイムで計算・表示
- 出金申請ボタンを押下後、完了画面へ遷移

## 4. API連携（画面表示に必要なデータ）

### 4.1 入金履歴取得API

レスポンス項目：

- 入金ID
- 日時
- 通貨（BTC/ETH）
- 入金額
- ステータス（処理中 / 完了）

### 4.2 出金申請API

リクエスト項目：

- 通貨
- 出金先アドレス
- 出金額

レスポンス：

- 成功 / エラーメッセージ

## 5. スケジュール

- 設計：2週間
- 実装：4週間
- テスト：2週間

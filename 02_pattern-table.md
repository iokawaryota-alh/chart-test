# パターン表埋め

テスト設計の第2段階：因子と水準を組み合わせて、具体的なテストパターンを作成する作業です。

## 目的

- 因子水準出しで洗い出した要素を組み合わせて、テストケースのパターンを作る
- 正常系、異常系、境界値を網羅的にカバーする
- 組み合わせ爆発を避けるため、効率的な絞り込みを行う

## パターン表とは

因子を列、水準を各行の値として、テストケースを表形式で整理したもの。

例：

| No  | 通貨 | 出金先アドレス | 出金額 | 期待結果 |
| --- | ---- | -------------- | ------ | -------- |
| 1   | BTC  | 正常           | 正常   | 成功     |
| 2   | BTC  | 未入力         | 正常   | エラー   |
| 3   | BTC  | 形式不正       | 正常   | エラー   |
| 4   | BTC  | 正常           | 未入力 | エラー   |
| ... | ...  | ...            | ...    | ...      |

## 作業の流れ

1. 因子水準表を元に、テストパターンの列を決める
2. 正常系のパターンを作成（全て正常な値の組み合わせ）
3. 異常系のパターンを作成（1つずつ異常値に変える）
4. 境界値のパターンを追加
5. 組み合わせテスト技法（ペアワイズなど）で効率化を検討
6. 期待結果（OK/NG、エラーメッセージなど）を記入

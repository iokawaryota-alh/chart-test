# 暗号資産取引デモサイト - アプリケーション構成

## 概要

テスト設計の演習用デモサイトです。
暗号資産（BTC、ETH）の取引・入出金機能を模擬したWebアプリケーションで、プロフェッショナルな取引所風のUIを採用しています。

**v2.0.0より、Bybit APIを使用した実取引所データの表示と、本格的な注文機能を実装しました。**

銘柄追加時は、次の3点をセットで更新すると漏れが起きにくいです。

最後に `pytest test/ -v` を実行し、チャート表示・注文作成・資産状況の回帰を確認してください。

### 主要コンポーネント

#### 1. バックエンド（server.py）

- **Flask** - Pythonウェブフレームワーク
- **CORS** - クロスオリジンリクエスト対応
- **REST API** - 以下のエンドポイントを提供：

  **資産関連**
  - `GET /api/balance` - 残高取得（BTC, ETH, JPY）
  - `GET /api/deposits` - 入金履歴取得
  - `GET /api/withdrawals` - 出金履歴取得
  - `POST /api/withdraw` - 出金申請

  **注文関連（v2.0.0で追加）**
  - `GET /api/current-price` - 現在のBTC/JPY価格取得
  - `POST /api/orders` - 新規注文作成（成行・指値）
  - `GET /api/orders?status={status}` - 注文一覧取得（open/filled/all）
  - `DELETE /api/orders/{order_id}` - 注文キャンセル

  **管理者API（v2.1.0で追加）**
  - `POST /api/admin/login` - 管理者ログイン
  - `GET /api/admin/settings` - システム設定取得
  - `PUT /api/admin/settings` - システム設定更新
  - `GET /api/admin/balance` - 残高情報取得
  - `PUT /api/admin/balance` - 残高情報更新
  - `GET /api/admin/orders` - 注文管理一覧
  - `GET /api/admin/stats` - 統計情報取得
  - `POST /api/admin/orders/{order_id}/fill` - 注文約定
  - `DELETE /api/admin/orders/{order_id}/cancel` - 注文キャンセル

#### 2. フロントエンド（React）

- **React 18** - UIライブラリ（CDNバージョン）
- **Babel Standalone** - JSX変換
- **Chart.js 4.4.1** - チャート描画（v2.0.0でLightweight Chartsから移行）
- **date-fns adapter** - 時系列データ対応
- **Bybit API** - リアルタイム価格データ（v2.0.0で追加）
  - WebSocket: `wss://stream.bybit.com/v5/public/linear`
  - REST API: `https://api.bybit.com/v5/market`

#### 3. 主要Reactコンポーネント

- **App** - メインアプリケーションコンポーネント
- **SideMenu** - 左サイドメニュー（6項目）
- **PriceChart** - BTC/JPY価格チャート（Bybit API連携、WebSocketリアルタイム更新）
- **TradingPanel** - 注文パネル（成行・指値注文、v2.0.0で追加）
- **OrderBook** - オーダーブック表示
- **RecentTrades** - 最近の取引一覧
- **HistoryView** - 取引履歴表示（注文履歴、入出金履歴）
- **DepositTab** - 入金タブ
- **WithdrawTab** - 出金タブ

#### 4. スタイリング

- **取引所風ダークテーマ** - プロフェッショナルなUI
- **カラースキーム**:
  - 背景: `#1a1b1e`, `#0f1012`

## ✨ appでできること

### 1. リアルタイムチャート機能（v2.0.0で大幅強化）

- **実取引所データ表示** - Bybit APIからBTC/USDTの実データを取得
- **WebSocketリアルタイム更新** - 価格変動をリアルタイムで反映
- **時間足切替** - 1分 / 5分 / 15分 / 30分 / 1時間 / 4時間 / 1日
- **チャートタイプ切替** - ラインチャート / キャンドルスティックチャート
- **価格変動表示** - 期間内の変動率を表示
- **USD/JPY換算** - USDTから円への自動換算（レート150）
- **接続状態表示** - WebSocket接続状況を色で表示（●）

### 2. 注文機能（v2.0.0で新規追加）

#### 注文種別

- **買い注文（Buy）** - JPY残高を使用してBTCを購入
- **売り注文（Sell）** - BTC残高を売却してJPYを取得

#### 注文タイプ

- **成行注文（Market Order）** - 現在価格で即座に約定
- **指値注文（Limit Order）** - 指定価格での注文（未約定として保持）

#### 注文機能の詳細

- **現在価格表示** - リアルタイム更新される現在のBTC/JPY価格
- **Quick Amountボタン** - ワンクリックで残高の25%/50%/75%/100%を自動計算
- **リアルタイム計算**
  - 概算金額（価格 × 数量）
  - 手数料（0.1%）
  - 合計金額（手数料込み/手数料引き）
- **包括的なバリデーション**
  - 最小注文数量：0.0001 BTC
  - 指値価格範囲：現在価格の±10%以内
  - 残高チェック（買い：JPY、売り：BTC）
  - 数値形式チェック
- **エラー表示** - 8種類の詳細なエラーメッセージ
- **成功通知** - 注文完了時の成功メッセージ

### 3. 注文管理機能（v2.0.0で新規追加）

- **注文履歴タブ**
  - 全注文の一覧表示（日時、売買、種別、価格、数量、ステータス）
  - ステータス別バッジ表示
    - 約定（filled）：緑色バッジ
    - 未約定（open）：黄色バッジ
    - キャンセル済（cancelled）：灰色バッジ
  - 未約定注文のキャンセル機能

### 4. 入金機能

- **入金用アドレス表示** - BTC/ETHの入金アドレス
- **入金履歴表示** - 過去の入金記録を一覧表示
- **ステータス表示** - 完了/処理中のバッジ表示

### 5. 出金機能

- **通貨選択** - BTC / ETH のラジオボタン選択
- **出金先アドレス入力** - ウォレットアドレスの入力
- **出金額入力** - 任意の金額を指定
- **手数料自動計算** - 通貨ごとの手数料を自動計算
  - BTC: 0.0005 BTC
  - ETH: 0.005 ETH
- **受取実額表示** - 手数料を差し引いた受取額を表示
- **出金履歴表示** - 過去の出金申請を一覧表示

### 6. バリデーション機能

#### 出金バリデーション

- **必須チェック** - アドレス・金額の入力必須
- **アドレス形式チェック**
  - BTC: `1`、`3`、または`bc1`で始まる文字列
  - ETH: `0x`で始まる42文字
- **残高不足チェック** - 出金額が残高を超えないか検証
- **最小金額チェック**
  - BTC: 0.001 BTC
  - ETH: 0.01 ETH

#### 注文バリデーション（v2.0.0で追加）

- **数量チェック**
  - 最小注文数量：0.0001 BTC
  - 正の数値のみ許可
  - 数値形式検証
- **価格チェック（指値注文）**
  - 現在価格の±10%以内に制限
  - 正の数値のみ許可
- **残高チェック**
  - 買い注文：JPY残高（手数料0.1%込み）
  - 売り注文：BTC残高
- **詳細エラーメッセージ**
  - 不足金額や必要残高を具体的に表示
- **最小金額チェック**
  - BTC: 0.0001 BTC以上
  - ETH: 0.001 ETH以上

### 7. UI/UX機能

- **サイドメニュー** - 6つの主要機能へのナビゲーション
- **タブ切替** - 取引履歴（注文履歴/入出金履歴）
- **残高表示** - BTC/ETH/JPY残高を常時表示
- **純資産額表示** - 円換算での合計資産額表示
- **レスポンシブデザイン** - グリッドレイアウトで柔軟な表示
- **ローディング表示** - 処理中のスピナー表示（注文処理時）
- **エラー表示** - バリデーションエラーの詳細表示
- **成功通知** - 注文完了時の成功メッセージ
- **ダークテーマ** - プロフェッショナルな取引所風UI

## 🎯 テスト設計演習としての価値

このデモサイトは、以下のテスト設計スキルを実践的に学習できるように設計されています：

### 1. 入力バリデーションテスト

- **境界値分析**: 最小注文数量（0.0001 BTC）、価格範囲（現在価格±10%）
- **同値分割**: 有効/無効な数量、有効/無効な価格
- **異常系テスト**: 文字列入力、負数、ゼロ、NULL値

### 2. エラーメッセージ設計

- **8種類の詳細エラーメッセージ**
  - 注文種別エラー
  - 数量エラー（未入力、形式、範囲、最小値）
  - 価格エラー（未入力、形式、範囲）
  - 残高不足エラー（具体的な不足額表示）
- **エラー表示位置**: 注文パネル内にインライン表示
- **エラー優先順位**: 必須チェック → 形式チェック → 範囲チェック

### 3. 状態遷移テスト

- **注文ステータス遷移**
  - 未約定（open）→ 約定（filled）
  - 未約定（open）→ キャンセル（cancelled）
- **残高の変化パターン**
  - 買い注文約定時：JPY減少、BTC増加
  - 売り注文約定時：BTC減少、JPY増加
- **画面遷移**: 注文パネル → 取引履歴 → 注文詳細

### 4. 組み合わせテスト

- **組み合わせ要素**
  - 注文種別：買い × 売り
  - 注文タイプ：成行 × 指値
  - 数量：有効範囲 × 境界値 × 異常値
  - 価格：有効範囲 × 境界値 × 異常値
- **推奨テスト技法**: ペアワイズ法、オールペア法

### 5. データ整合性テスト

- **残高整合性**: 注文前後の残高計算が正確か
- **手数料計算**: 0.1%の手数料が正しく計算されるか
- **注文履歴**: 注文データが正確に記録されるか
- **チャートデータ**: WebSocketでのリアルタイム更新が正確か

### 6. 非機能テスト観点

- **パフォーマンス**: WebSocket接続遅延、API応答時間
- **可用性**: WebSocket切断時の自動再接続
- **ユーザビリティ**: エラーメッセージの理解しやすさ、Quick Amountボタンの操作性

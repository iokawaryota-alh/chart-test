# 暗号資産取引デモサイト - アプリケーション構成

## 概要

テスト設計の演習用デモサイトです。
暗号資産（BTC、ETH）の取引・入出金機能を模擬したWebアプリケーションで、プロフェッショナルな取引所風のUIを採用しています。

---

## 🏗️ appを構成する要素

### ディレクトリ構造

```
app/
├── server.py                    # Flaskバックエンドサーバー
├── static/                      # 静的ファイル
│   ├── index.html              # メインHTMLファイル
│   ├── css/
│   │   └── style.css           # 取引所風ダークテーマスタイル
│   └── js/
│       └── app.js              # React アプリケーションロジック
└── README.md                    # このファイル
```

### 主要コンポーネント

#### 1. バックエンド（server.py）

- **Flask** - Pythonウェブフレームワーク
- **CORS** - クロスオリジンリクエスト対応
- **REST API** - 以下のエンドポイントを提供：
  - `GET /api/balance` - 残高取得
  - `GET /api/deposits` - 入金履歴取得
  - `GET /api/withdrawals` - 出金履歴取得
  - `POST /api/withdraw` - 出金申請

#### 2. フロントエンド（React）

- **React 18** - UIライブラリ（CDNバージョン）
- **Babel Standalone** - JSX変換
- **Chart.js 4.4.1** - チャート描画
- **date-fns adapter** - 時系列データ対応

#### 3. 主要Reactコンポーネント

- **App** - メインアプリケーションコンポーネント
- **PriceChart** - BTC/JPY価格チャート（CoinGecko API連携）
- **DepositTab** - 入金タブ
- **WithdrawTab** - 出金タブ

#### 4. スタイリング

- **取引所風ダークテーマ** - プロフェッショナルなUI
- **カラースキーム**:
  - 背景: `#1a1b1e`, `#0f1012`
  - アクセント: `#4a9eff`
  - ボーダー: `#2a2b2e`, `#3a3b3e`
  - テキスト: `#e8e8e8`, `#888`

## ✨ appでできること

### 1. 価格チャート機能

- **リアルタイム価格表示** - CoinGecko APIからBTC/JPY価格を取得
- **期間切替** - 1H / 1D / 1W / 1M
- **価格変動表示** - 期間内の変動率を表示
- **インタラクティブチャート** - Chart.jsによる滑らかな描画

### 2. 入金機能

- **入金用アドレス表示** - BTC/ETHの入金アドレス
- **入金履歴表示** - 過去の入金記録を一覧表示
- **ステータス表示** - 完了/処理中のバッジ表示

### 3. 出金機能

- **通貨選択** - BTC / ETH のラジオボタン選択
- **出金先アドレス入力** - ウォレットアドレスの入力
- **出金額入力** - 任意の金額を指定
- **手数料自動計算** - 通貨ごとの手数料を自動計算
  - BTC: 0.0005 BTC
  - ETH: 0.005 ETH
- **受取実額表示** - 手数料を差し引いた受取額を表示
- **出金履歴表示** - 過去の出金申請を一覧表示

### 4. バリデーション機能

- **必須チェック** - アドレス・金額の入力必須
- **アドレス形式チェック**
  - BTC: `1`または`3`で始まる26-35文字
  - ETH: `0x`で始まる42文字
- **残高不足チェック** - 出金額が残高を超えないか検証
- **最小金額チェック**
  - BTC: 0.0001 BTC以上
  - ETH: 0.001 ETH以上

### 5. UI/UX機能

- **タブ切替** - チャート/入金/出金の3タブ
- **残高表示** - サイドバーにBTC/ETH残高を常時表示
- **純資産額表示** - ヘッダーに合計資産額を表示
- **レスポンシブデザイン** - グリッドレイアウトで柔軟な表示
- **ローディング表示** - 処理中のスピナー表示
- **エラー表示** - バリデーションエラーのハイライト表示

## テスト設計の演習ポイント

このデモサイトを使って、以下のテスト設計を練習できます：

1. **因子水準出し**
   - 入力項目（通貨、アドレス、金額）
   - 各項目の水準（正常値、異常値、境界値など）

2. **パターン表作成**
   - 正常系、異常系の組み合わせ
   - バリデーションエラーのパターン

3. **手順・期待値の記述**
   - 具体的な操作手順
   - エラーメッセージの確認
   - 画面遷移の確認
